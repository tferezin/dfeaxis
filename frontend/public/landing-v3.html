E o recebimento fiscal?

Em dezembro de 2025, a SAP encerrou o suporte ao GRC NF-e. A pesquisa Tax do Amanhã 2025 da Deloitte revelou que 66% das empresas demonstraram receio com a fase de transição. Milhares migraram para o DRC — e descobriram que a automação do recebimento de documentos fiscais de fornecedores não veio junto.

Hoje, equipes fiscais de empresas com SAP ainda acessam a SEFAZ manualmente, baixam XMLs de NF-e e CT-e um a um, e correm o risco de perder créditos de ICMS sobre frete por falta de captura tempestiva. Quando a SEFAZ fica instável, ninguém sabe quais documentos ficaram pra trás. E quando o auditor pede rastreabilidade do processo, a resposta é um print de tela.

Com a Reforma Tributária exigindo rastreabilidade em tempo real de créditos e débitos — IBS com integração instantânea entre todos os elos da cadeia, CBS com rastreabilidade ininterrupta em nível transacional — esse processo manual virou bomba-relógio.

O DFeAxis resolve exatamente esse gap: captura automática de documentos fiscais de fornecedores na SEFAZ, com entrega direta ao SAP DRC — sem middleware, sem intervenção humana.

Nenhum compromisso.

Certificado A1 cifrado

O .pfx é processado em memória RAM e nunca gravado em disco em texto claro. Cifrado com AES-256 usando chave derivada do CNPJ + secret exclusivo do tenant.

02

Resiliência com circuit breaker

Circuit breaker · retry · backoff
03

Controle de NSU sem gaps

A sequência de NSUs por CNPJ é rastreada continuamente. Se a SEFAZ retornar fora de sequência, o gap detector agenda reprocessamento antes de confirmar qualquer entrega.

NSU tracking · dead letter queue

Dúvidas frequentes

Perguntas que todo
time fiscal faz.

Precisa de SAP PI ou SAP Integration Suite?

Não. O DFeAxis se conecta ao SAP DRC via RFC Destination HTTP padrão (SM59). Sem SAP PI, sem CPI, sem cenário B2B, sem middleware adicional.

Funciona com SAP DRC Cloud (Public Cloud)?

Sim. O DFeAxis expõe uma API REST pública acessível por qualquer ambiente SAP com acesso à internet via Communication Arrangement ou RFC Destination HTTP.

Migramos do SAP GRC NFe para o DRC e perdemos a automação. O DFeAxis resolve?

Exatamente. O GRC fazia polling automático via SAP PI (cenário CFB2B) a cada 15–30 min. O DRC não tem esse mecanismo nativo. O DFeAxis foi criado especificamente para resolver essa lacuna — polling automático, circuit breaker contra erro 656 e controle de NSU sem gaps.

Meus XMLs ficam armazenados em servidores de terceiros?

Não. O DFeAxis opera com zero-retention por design. O XML é capturado na SEFAZ, entregue ao seu ERP via API REST e descartado imediatamente após a confirmação. Seus documentos fiscais não ficam armazenados nos nossos servidores além do tempo estritamente necessário para a entrega — normalmente segundos.

O certificado digital A1 fica seguro?

Sim. O .pfx é cifrado com AES-256 com chave exclusiva derivada por tenant. Nunca é gravado em disco em texto claro. Isolamento total — nenhum dado de uma empresa é acessível por outra.

Posso testar por 10 dias sem cartão?

Sim. Nenhum cartão é solicitado no cadastro. Ao final dos 10 dias o sistema para automaticamente. O modelo é créditos pré-pagos — você compra quando e quanto quiser, sem cobranças automáticas.

Funciona apenas com SAP ou integra com outros ERPs?

O SAP DRC é a integração principal, mas o DFeAxis expõe API REST padrão compatível com qualquer ERP (TOTVS, Oracle, Senior, Sankhya) ou sistema próprio que faça chamadas HTTP.

Funciona com TOTVS / Oracle / outros ERPs?

Sim. O DFeAxis expõe API REST padrão. Qualquer sistema que consuma REST pode integrar. Para SAP DRC temos integração nativa via RFC Destination HTTP. Para TOTVS, Oracle, Senior, Sankhya e outros, basta consumir os endpoints REST.

Preciso de middleware (SAP PI, CPI, MuleSoft) para integrar?

Não. O DFeAxis se conecta diretamente ao seu ERP via API REST. Sem SAP PI, sem CPI, sem MuleSoft, sem cenários B2B. Para SAP DRC, basta configurar uma RFC Destination HTTP na SM59.
